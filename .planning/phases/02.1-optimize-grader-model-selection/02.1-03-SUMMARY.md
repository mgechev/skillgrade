## Decisions Made
- **qwen2.5:3b as default model:** The only model among 8 candidates with perfect discrimination (positive=1.0, empty=0.0, wrong=0.0), consistent across all 3 benchmark profiles. ~4.6s median wall time.
- **JSON Schema format unconditional:** All models in the benchmark supported json_schema mode with 100% validity. No need for a no-schema fallback.
- **Hardcoded params over config:** num_ctx=8192, num_predict=512, timeout=60s are internal Ollama concerns. Exposing them through GraderConfig was an abstraction leak.
- **default profile preferred:** The optimized-all profile (OLLAMA_FLASH_ATTENTION=1, KV_CACHE_TYPE=q4_0) actually slowed models down. No special env vars needed.
