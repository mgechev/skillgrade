<task type="auto">
  <name>Task 1: Create CI benchmark workflow with Ollama setup and model caching</name>
  <files>.github/workflows/benchmark-grader.yml</files>
  <action>
Create `.github/workflows/benchmark-grader.yml` with the following structure:

**Triggers:**
```yaml
on:
  workflow_dispatch:
    inputs:
      models:
        description: 'Comma-separated model tags to benchmark'
        required: false
        default: 'phi3.5:3.8b-mini-instruct-q4_K_M,qwen2.5:3b,llama3.2:3b,gemma3:4b,qwen2.5:7b'
      profile:
        description: 'Tuning profile: default, optimized-env, optimized-all, or all'
        required: false
        default: 'all'
  pull_request:  # Temporary during Phase 2.1 development -- REMOVE before phase completes
```

**Workflow-level configuration:**
- `name: Benchmark Grader Models`
- `permissions: contents: read`
- `concurrency` group keyed to workflow + PR/ref with cancel-in-progress (same pattern as ci.yml)
- `env: NODE_OPTIONS: '--max-old-space-size=4096'`

**Job: benchmark**
- `runs-on: ubuntu-24.04-arm`
- `timeout-minutes: 60` (72 runs x ~30s = ~36 min, plus model pull time)

**Steps:**

1. `actions/checkout@v4`

2. `./.github/actions/setup-node` (existing composite action)

3. `ai-action/setup-ollama@v2` with `version: '0.17.7'`
   - Add a comment: `# ARM64 support: downloads ollama-linux-arm64 binary`

4. **Cache Ollama models** using `actions/cache@v5`:
   ```yaml
   - uses: actions/cache@v5
     with:
       path: ~/.ollama
       key: ollama-models-${{ inputs.models || 'default-set' }}
       restore-keys: |
         ollama-models-
   ```

5. **Start Ollama with default profile** (no optimized env vars):
   ```yaml
   - name: Start Ollama (default profile)
     run: |
       ollama serve &
       for i in $(seq 1 30); do
         curl -sf http://localhost:11434/ > /dev/null 2>&1 && break
         sleep 1
       done
       echo "[OK] Ollama ready"
   ```

6. **Pull candidate models:**
   ```yaml
   - name: Pull models
     run: |
       IFS=',' read -ra MODELS <<< "${{ inputs.models || 'phi3.5:3.8b-mini-instruct-q4_K_M,qwen2.5:3b,llama3.2:3b,gemma3:4b,qwen2.5:7b' }}"
       for model in "${MODELS[@]}"; do
         echo "[INFO] Pulling $model..."
         ollama pull "$model"
       done
   ```

7. **Run benchmark with default profile:**
   ```yaml
   - name: Benchmark (default profile)
     if: ${{ inputs.profile == 'default' || inputs.profile == 'all' || inputs.profile == '' }}
     run: |
       MODELS_ARG=""
       if [ -n "${{ inputs.models }}" ]; then
         MODELS_ARG="--models ${{ inputs.models }}"
       fi
       npx ts-node tests/benchmark-grader.ts --profile default --output benchmark-results/ci-default.json $MODELS_ARG
   ```

8. **Stop Ollama, restart with optimized env vars:**
   ```yaml
   - name: Restart Ollama (optimized profile)
     if: ${{ inputs.profile == 'optimized-env' || inputs.profile == 'optimized-all' || inputs.profile == 'all' || inputs.profile == '' }}
     run: |
       pkill -f "ollama serve" || true
       sleep 2
       OLLAMA_FLASH_ATTENTION=1 \
       OLLAMA_KV_CACHE_TYPE=q8_0 \
       OLLAMA_NUM_PARALLEL=1 \
       OLLAMA_NUM_THREAD=4 \
       ollama serve &
       for i in $(seq 1 30); do
         curl -sf http://localhost:11434/ > /dev/null 2>&1 && break
         sleep 1
       done
       echo "[OK] Ollama ready (optimized)"
   ```

9. **Run benchmark with optimized-env profile:**
   ```yaml
   - name: Benchmark (optimized-env profile)
     if: ${{ inputs.profile == 'optimized-env' || inputs.profile == 'all' || inputs.profile == '' }}
     run: |
       MODELS_ARG=""
       if [ -n "${{ inputs.models }}" ]; then
         MODELS_ARG="--models ${{ inputs.models }}"
       fi
       npx ts-node tests/benchmark-grader.ts --profile optimized-env --output benchmark-results/ci-optimized-env.json $MODELS_ARG
   ```

10. **Run benchmark with optimized-all profile:**
    ```yaml
    - name: Benchmark (optimized-all profile)
      if: ${{ inputs.profile == 'optimized-all' || inputs.profile == 'all' || inputs.profile == '' }}
      run: |
        MODELS_ARG=""
        if [ -n "${{ inputs.models }}" ]; then
          MODELS_ARG="--models ${{ inputs.models }}"
        fi
        npx ts-node tests/benchmark-grader.ts --profile optimized-all --output benchmark-results/ci-optimized-all.json $MODELS_ARG
    ```

11. **Upload results as artifact:**
    ```yaml
    - uses: actions/upload-artifact@v4
      if: always()
      with:
        name: benchmark-results
        path: benchmark-results/
        retention-days: 30
    ```

**Notes for the executor:**
- The `pull_request` trigger is deliberately included during Phase 2.1 development. A later plan will remove it before the phase completes.
- Use `if: always()` on the artifact upload so results are saved even if a benchmark step fails.
- The model list for `pull_request` runs defaults to the 5 primary models (one per family, default quant). The full 8-quant matrix is used for `workflow_dispatch`.
- `OLLAMA_NUM_THREAD=4` is hardcoded for CI (4 vCPU runner). The local wrapper script uses 12 for Snapdragon.
- Follow existing ci.yml patterns for permissions, concurrency, env.
