**2. Add Ollama configuration warning to src/graders/index.ts:**

Add a private method `warnOllamaConfig()` to the `LLMGrader` class that checks `process.env` for optimal Ollama settings. Call it once from the `grade()` method when Ollama is available (after `checkOllamaAvailability` returns `available: true`, before calling `callOllamaWithRetry`).

```typescript
private warnedAboutConfig = false;

private warnOllamaConfig(): void {
    if (this.warnedAboutConfig) {
        return;
    }

    this.warnedAboutConfig = true;

    const warnings: string[] = [];

    if (!process.env.OLLAMA_FLASH_ATTENTION) {
        warnings.push('OLLAMA_FLASH_ATTENTION not set -- flash attention disabled');
    }

    if (!process.env.OLLAMA_KV_CACHE_TYPE) {
        warnings.push('OLLAMA_KV_CACHE_TYPE not set -- using FP16 KV cache (higher RAM usage)');
    }

    if (warnings.length > 0) {
        console.warn(`[LLMGrader] Suboptimal Ollama configuration detected (set env vars before starting "ollama serve"):`);

        for (const w of warnings) {
            console.warn(`  - ${w}`);
        }
    }
}
```

Key design decisions:
- `warnedAboutConfig` flag ensures the warning prints only once per LLMGrader instance (not per grade() call)
- This is a warning, NOT a failure -- grade() continues regardless
- Only checks the two env vars that have the highest performance impact per research
- The warning message tells the user when to set them ("before starting ollama serve")
- Per research: this is best-effort -- it checks the Node.js process environment, not the Ollama server's actual configuration. If Ollama was started separately (common on Windows desktop), these checks may produce false warnings. This limitation is acceptable per the user's decision ("warns, not fails").

Call placement in `grade()`:
```typescript
if (ollamaStatus.available) {
    this.warnOllamaConfig();  // <-- Add here
    const ollamaResult = await this.callOllamaWithRetry(prompt, ollamaHost, config);
    // ...
}
```

## Ollama Configuration

For optimal performance with the local LLM grader, set these environment variables **before** starting `ollama serve`:

OLLAMA_FLASH_ATTENTION=1 OLLAMA_KV_CACHE_TYPE=q8_0 OLLAMA_NUM_PARALLEL=1 OLLAMA_NUM_THREAD=$(nproc) ollama serve

$env:OLLAMA_FLASH_ATTENTION="1"; $env:OLLAMA_KV_CACHE_TYPE="q8_0"; $env:OLLAMA_NUM_PARALLEL="1"; $env:OLLAMA_NUM_THREAD="12"; ollama serve
```
