Create `.github/workflows/skill-eval.yml` with the following structure per user decisions:

**Workflow metadata:**
- `name: Skill Eval`
- Triggers: `pull_request`, `push` to `[main]`, `workflow_dispatch`
- `permissions: contents: read`
- Concurrency: `group: ${{ github.workflow }}-${{ github.event.pull_request.number || github.ref }}`, `cancel-in-progress: true`
- `env: NODE_OPTIONS: '--max-old-space-size=4096'`

**Job 1: eval-local**
- `name: Eval (local)`
- `runs-on: ubuntu-24.04-arm`
- `timeout-minutes: 30`
- Steps:
  1. `uses: actions/checkout@v4`
  2. `uses: ./.github/actions/setup-node`
  3. `uses: ./.github/actions/setup-ollama`
  4. `name: Run evaluation` / `run: npm run validate -- superlint_demo --provider=local`
  5. `name: Preview results` / `if: always()` / `run: npm run preview`
  6. `uses: actions/upload-artifact@v4` / `if: always()` / `with: name: eval-results-local, path: results/`

**Job 2: eval-docker**
- `name: Eval (docker)`
- `runs-on: ubuntu-24.04-arm`
- `timeout-minutes: 30`
- Steps:
  1. `uses: actions/checkout@v4`
  2. `uses: ./.github/actions/setup-node`
  3. `uses: ./.github/actions/setup-ollama`
  4. Docker image caching (compute cache key, restore, load):
     ```yaml
     - name: Compute Docker cache key
       id: docker-cache
       run: |
         HASH=$(find tasks/superlint_demo -type f | sort | xargs sha256sum | sha256sum | cut -c1-16)
         echo "key=docker-eval-superlint-$HASH" >> "$GITHUB_OUTPUT"
     - name: Restore Docker image cache
       id: docker-restore
       uses: actions/cache@v5
       with:
         path: /tmp/docker-cache
         key: ${{ steps.docker-cache.outputs.key }}
     - name: Load cached Docker image
       if: steps.docker-restore.outputs.cache-hit == 'true'
       run: docker load -i /tmp/docker-cache/eval-image.tar
     ```
  5. `name: Run evaluation` / `run: npm run validate -- superlint_demo --provider=docker`
  6. Docker image save (only on cache miss):
     ```yaml
     - name: Save Docker image to cache
       if: steps.docker-restore.outputs.cache-hit != 'true'
       run: |
         mkdir -p /tmp/docker-cache
         IMAGE=$(docker images --format '{{.Repository}}:{{.Tag}}' | grep skill-eval-superlint | head -1)
         if [ -n "$IMAGE" ]; then
           docker save -o /tmp/docker-cache/eval-image.tar "$IMAGE"
         fi
     ```
  7. `name: Preview results` / `if: always()` / `run: npm run preview`
  8. `uses: actions/upload-artifact@v4` / `if: always()` / `with: name: eval-results-docker, path: results/`

Key details:
- Docker is pre-installed on `ubuntu-24.04-arm` runners (no setup needed).
- Docker cache key is computed from task file content hash -- deterministic, auto-invalidates on task changes.
- Cache save only runs on cache miss (`cache-hit != 'true'`), load only on hit.
- Content-hash image naming from Quick Task 3 means the `grep skill-eval-superlint` pattern finds the correct image.
- Both jobs run in parallel (no `needs:` dependency between them).
- Each job calls setup-ollama independently (separate runner VMs).
- No explicit `retention-days` on artifact upload (default retention per user decision).
