# Plan 05.1-01 Summary: Config Foundation + Screening + Full Validation

## Outcome

**COMPLETE** -- qwen3:1.7b with prompt prefix fix achieves reward >= 0.97 on both Docker and Local providers, validated with 3 trials each. The original Phase 5.1 goal (reward >= 0.85) is exceeded. Plans 02 and 03 are superseded -- no further tuning or Docker-specific validation needed.

## What Was Done

### 1. Config and code changes (Task 1)

- **Model switch**: qwen3.5:4b -> qwen3:1.7b (2x faster, half memory, better /no_think compliance)
- **Tool denial**: 8 tools denied (glob, grep, list, task, todowrite, skill, webfetch, websearch, codesearch, external_directory), 4 allowed (bash, read, edit, write) + doom_loop
- **Prompt prefix**: Added `description` field hint for bash tool calls -- the single biggest fix
- **Env flags**: OPENCODE_DISABLE_CLAUDE_CODE_PROMPT, OPENCODE_DISABLE_PROJECT_CONFIG, OPENCODE_DISABLE_EXTERNAL_SKILLS added to opencode run command
- **Modelfile**: qwen3-1.7b-opencode-agent with num_ctx 8192, num_predict 4096, temperature 0, num_batch 1024, num_thread 8

### 2. Screening experiments (Task 2)

Earlier sessions ran 4 screening experiments (documented in experiments/01-04) covering:
- Qwen 3.5 bug confirmation (tool calls printed as text, not executed)
- Qwen3:4b screening (partial success, then OOM under memory pressure)
- Memory/throughput matrix (qwen3:1.7b identified as sweet spot)
- Prompt eval bottleneck analysis (70% of Ollama time)

### 3. Docker breakthrough

Docker provider eliminates SIGSEGV entirely (native linux-arm64 on WSL2). Combined with the prompt prefix fix (`description` field hint), first validated trial achieved reward 0.97 in 374s.

## Impact Ranking

1. **Prompt prefix with `description` field hint** -- reward 0.00 -> 0.97. qwen3:1.7b omits the required `description` field in bash tool calls without this hint. One line fixed everything.
2. **Model switch to qwen3:1.7b** -- 2x faster (50 vs 22 tok/s), half memory (1.7 vs 3.2 GB), better /no_think compliance (375 vs 918 tokens). Smaller model, better results.
3. **OPENCODE_DISABLE_* env flags** -- keeps context window clean. Local provider uses fewer commands (6 vs 8) and half the output tokens (220 vs 440) compared to Docker, likely due to cleaner prompt.
4. **Tool denial config** -- reduces cognitive load. Denied tools still get descriptions sent (opencode limitation), but permission denial prevents wasted turns.
5. **Docker provider** -- eliminates SIGSEGV for CI. On local, OPENCODE_BIN_PATH to ARM64 dev binary achieves the same.

## Decisions

| Decision | Rationale | Outcome |
