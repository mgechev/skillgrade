**Docker-specific config adjustment:**

The Docker provider runs commands inside a container. Inside the container, `localhost` refers to the container itself, not the host. The opencode.json config must use `host.docker.internal:11434/v1` instead of `localhost:11434/v1` for the Ollama baseURL.

**Implementation approach:** Modify `OpenCodeAgent.run()` to detect whether the workspace is a Docker context and adjust the config accordingly. Detection heuristic: Docker workspaces typically have paths like `/workspace/...` while local workspaces have platform-specific paths. Alternatively, a simpler approach: always write the config with a placeholder, then use `sed` via runCommand to replace the host based on context.

**Recommended approach:** In `OpenCodeAgent.run()`, after reading the static config JSON:
1. Parse the JSON
2. Check if the workspace path looks like a Docker container path (starts with `/workspace` or `/tmp` on a non-Linux host, OR check if `runCommand('hostname')` returns a Docker container ID)
3. If Docker detected: replace `localhost` with `host.docker.internal` in the baseURL
4. Write the adjusted JSON to the workspace

A simpler alternative: just write the config and test. If the Docker container can reach `localhost:11434` (some Docker configs do route this), no change needed. Windows Docker Desktop with WSL2 backend typically resolves `host.docker.internal` automatically.

**Also ensure:**
- opencode binary must be available inside the Docker container. Per user's locked decision: "opencode installed inside Docker container, matching the GeminiAgent/ClaudeAgent pattern." This means the Dockerfile (used by DockerProvider) must have opencode installed. Check if the existing Dockerfile already installs CLI tools. If not, the install command inside the container via runCommand may be needed: `runCommand('npm install -g @anthropic-ai/opencode')` or `runCommand('curl -fsSL https://opencode.ai/install | bash')`.
- Verify OLLAMA_HOST is set to 0.0.0.0 on the host machine so Ollama accepts connections from Docker containers.

**Run the eval:**
```bash
npm run eval -- superlint_demo --agent=opencode --provider=docker --trials=1
```

**Troubleshooting (Docker-specific from RESEARCH.md):**

If ECONNREFUSED from opencode inside container:
- Set `OLLAMA_HOST=0.0.0.0` on host and restart Ollama
- Verify `host.docker.internal` resolves: `runCommand('ping -c 1 host.docker.internal')`
- If on Linux Docker (not Docker Desktop), may need ExtraHosts in container config

If opencode binary not found in container:
- Install opencode in the container before running: add a setup step via runCommand
- Or modify the Dockerfile to include opencode installation

**Fix any issues discovered.** Re-run until the trial completes with a scored result.
