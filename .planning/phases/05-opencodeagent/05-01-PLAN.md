```typescript
export class GeminiAgent extends BaseAgent {
    async run(
        instruction: string,
        _workspacePath: string,
        runCommand: (cmd: string) => Promise<CommandResult>
    ): Promise<string> {
        const b64 = Buffer.from(instruction).toString('base64');
        await runCommand(`echo '${b64}' | base64 -d > /tmp/.prompt.md`);
        const command = `gemini -y --sandbox=none -p "$(cat /tmp/.prompt.md)"`;
        const result = await runCommand(command);
        if (result.exitCode !== 0) {
            console.error('GeminiAgent: Gemini CLI failed to execute correctly.');
        }
        return result.stdout + '\n' + result.stderr;
    }
}
```

**Create `src/agents/opencode/index.ts`** -- OpenCodeAgent class following the GeminiAgent/ClaudeAgent pattern with three additions:

1. **Config injection:** Read the static JSON config file via `fs.readFileSync(path.join(__dirname, 'opencode.skill-eval-agent.json'), 'utf-8')`, base64-encode it, and write it to `opencode.json` in the workspace CWD via `runCommand`. This ensures opencode picks up the correct Ollama provider/model/permissions.

2. **Timeout wrapper:** Compute an internal timeout as 90% of `agent.timeout_sec` (passed from task.toml via the evalRunner). Since OpenCodeAgent does NOT have direct access to `task.toml`, hardcode a default timeout of 540 seconds (90% of 600s). Wrap the `opencode run` invocation with the bash `timeout` command: `timeout --signal=TERM --kill-after=10 ${timeoutSec} opencode run "$(cat /tmp/.prompt.md)"`. If `timeout` is unavailable (exit code 127), fall back to running opencode without the wrapper and log a warning. Exit code 124 from `timeout` means the process was killed -- log this.

3. **Model unload:** In a `finally` block after the run (same pattern as OllamaToolAgent), call `ollamaClient.chat({ model, messages: [], keep_alive: 0 })` to unload the model. Import `Ollama` from `'ollama'` and `DEFAULT_OLLAMA_AGENT_CONFIG` from `'../ollama/types'` for model name and host.

Additional:

- Use the established base64 prompt encoding pattern from GeminiAgent.
- The constructor creates the Ollama client instance (same as OllamaToolAgent).
- Export the class: `export class OpenCodeAgent extends BaseAgent`.
- For Docker variant (host.docker.internal), detect provider by checking if the workspace path starts with `/` and the config file source is not accessible via a direct filesystem path. Actually, since runCommand abstracts the provider, and opencode runs inside the container for Docker, the config needs `host.docker.internal` instead of `localhost`.
