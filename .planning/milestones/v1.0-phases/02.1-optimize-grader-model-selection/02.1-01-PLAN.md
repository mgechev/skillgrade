- `--models` (optional): comma-separated model tags. Default: all 8 model-quant combinations from research

- `--output` (optional): path for JSON results file. Default: `benchmark-results/<profile>.json`

Behavior:

        - Record: model, profile, fixture, run number, wall_time_ms, eval_count, eval_duration (ns), tokens_per_second, json_valid (boolean), score, reasoning (truncated to 100 chars), schema_mode ('json_schema' or 'no_schema')

  <action>
Create `scripts/ollama-bench.sh` -- a cross-platform bash script that orchestrates the three-point tuning matrix by managing Ollama server lifecycle with different environment variable profiles.

The script must:

1. Detect platform via `$OSTYPE` (msys/mingw for Git Bash on Windows, linux* for Linux)

2. Define `stop_ollama()` function:
   - Windows (msys/mingw): `taskkill.exe //IM ollama.exe //F 2>/dev/null || true` (double-slash for Git Bash flag escaping)
   - Linux: `pkill -f "ollama serve" 2>/dev/null || true`
   - Sleep 2 seconds after kill to allow port release

3. Define `wait_for_ollama()` function:
   - Poll `curl -sf http://localhost:11434/` with 1-second retries, max 30 retries
   - Print `[INFO] Waiting for Ollama...` on start, `[OK] Ollama ready` on success
   - Print `[ERROR] Ollama failed to start within 30s` and exit 1 on timeout

4. Define `start_ollama()` function that accepts env var arguments:
   - Stop any running Ollama first
   - Start `ollama serve` in background with specified env vars
   - Capture PID: `OLLAMA_PID=$!`
   - Call `wait_for_ollama()`

5. Run three profiles sequentially:

   **Profile 1: "default"** -- No Ollama env vars, default API params
   ```
   start_ollama (no extra env vars)
   npx ts-node tests/benchmark-grader.ts --profile default --output benchmark-results/default.json
   stop_ollama
   ```

   **Profile 2: "optimized-env"** -- Optimized env vars, default API params
   ```
   OLLAMA_FLASH_ATTENTION=1 \
   OLLAMA_KV_CACHE_TYPE=q8_0 \
   OLLAMA_NUM_PARALLEL=1 \
   start_ollama
   npx ts-node tests/benchmark-grader.ts --profile optimized-env --output benchmark-results/optimized-env.json
   stop_ollama
   ```
   Note: OLLAMA_NUM_THREAD is set based on platform: 12 for Windows/local (Snapdragon 12 cores), auto-detect via `nproc` on Linux.

   **Profile 3: "optimized-all"** -- Optimized env vars + optimized API params
   ```
   Same env vars as Profile 2
   npx ts-node tests/benchmark-grader.ts --profile optimized-all --output benchmark-results/optimized-all.json
   stop_ollama
   ```
   The benchmark script handles API-level param differences based on the profile name.

6. After all three profiles complete:
   - Print summary: `[OK] All profiles complete. Results in benchmark-results/`
   - Exit 0

7. Trap EXIT to ensure `stop_ollama` runs on script abort (Ctrl+C):
   ```
   trap 'stop_ollama; exit 1' INT TERM
   ```

8. Create `benchmark-results/` directory if it doesn't exist

9. Accept optional `--models` argument to pass through to the benchmark script (for testing with fewer models)

Use `#!/usr/bin/env bash` shebang. Use `set -euo pipefail` for strict mode. Use `[INFO]`/`[OK]`/`[WARN]`/`[ERROR]` prefixes for all console output (no emojis). Follow project shell conventions.

**Cross-platform notes from research:**
- Git Bash on Windows: `$OSTYPE` is `msys` or `mingw*`
- `taskkill.exe //IM` uses double-slash because Git Bash interprets single-slash as path prefix
- `ollama serve &` works in both Git Bash and Linux bash
- `nproc` is available on Linux but not Windows Git Bash; use hardcoded 12 for Windows (Snapdragon X Elite)
