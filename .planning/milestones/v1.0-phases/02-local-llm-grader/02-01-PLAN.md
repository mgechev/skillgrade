**Step 3: Implement in `src/graders/index.ts`**

Add three new private methods to `LLMGrader` class:

1. **`checkOllamaAvailability(ollamaHost: string, model: string): Promise<{ available: boolean; error?: string }>`**
   - `GET /` with `AbortSignal.timeout(5000)` for health check
   - If health fails with connection error: return `{ available: false, error: 'Ollama is not running at {host}. Start it with: ollama serve' }`
   - If health returns non-200: return `{ available: false, error: 'Ollama health check failed (HTTP {status})' }`
   - `GET /api/tags` to list models
   - Parse `models[]` array, check if requested model name matches (exact match or prefix match for models without tag)
   - If model not found: return `{ available: false, error: 'Ollama is running but model "{model}" is not pulled. Run: ollama pull {model}' }`
   - If all good: return `{ available: true }`

2. **`callOllama(prompt: string, ollamaHost: string, config: GraderConfig): Promise<GraderResult | null>`**
   - Model: `config.model || 'qwen3:4b'` (per user decision: same pattern as `config.model || 'gemini-2.0-flash'`)
   - POST to `{ollamaHost}/api/generate` with body:
     ```json
     {
       "model": model,
       "prompt": prompt,
       "stream": false,
       "format": {
         "type": "object",
         "properties": {
           "score": { "type": "number" },
           "reasoning": { "type": "string" }
         },
         "required": ["score", "reasoning"]
       },
       "options": {
         "temperature": 0,
         "num_predict": 512
       }
     }
     ```
   - Use `AbortSignal.timeout(300000)` (5 min timeout for generation)
   - If response not ok (e.g., 404 model not found): return null
   - Extract `data.response` text, pass to existing `this.parseResponse(text, config)`
   - On catch (connection error): return null

3. **`callOllamaWithRetry(prompt: string, ollamaHost: string, config: GraderConfig, maxRetries: number = 3): Promise<GraderResult | null>`**
   - Loop up to maxRetries attempts
   - Call `this.callOllama(prompt, ollamaHost, config)`
   - If null (connection error): return null immediately, do NOT retry
   - If result.score > 0 or last attempt: return result
   - If parse failure (score=0 with "Failed to parse"): wait `attempt * 1000` ms, retry
   - After all retries exhausted: return null

**Modify `grade()` method:**

Insert Ollama logic BEFORE the existing Gemini/Anthropic checks. The new flow:

```
1. Load rubric, build prompt (unchanged)
2. Determine ollamaHost: env?.OLLAMA_HOST || process.env.OLLAMA_HOST || 'http://localhost:11434'
3. Determine model: config.model || 'qwen3:4b'
4. Check Ollama availability: await this.checkOllamaAvailability(ollamaHost, model)
   - If available: try grading with callOllamaWithRetry()
     - If result (non-null): return result
     - If null: fall through to cloud (unexpected -- availability passed but call failed)
   - If not available AND no cloud API keys: return score 0 with the availability error message (fail fast per OLLAMA-01/02)
   - If not available AND cloud keys present: warn via console.warn and fall through (graceful degradation per OLLAMA-03)
5. Try Gemini (unchanged)
6. Try Anthropic (unchanged)
7. Return score 0 "No API key available" (updated message to mention Ollama too)
```

Update the final fallback error message from `'No API key available for LLM grading (set GEMINI_API_KEY or ANTHROPIC_API_KEY)'` to `'No LLM grading available (Ollama not running, no GEMINI_API_KEY or ANTHROPIC_API_KEY set)'`.
