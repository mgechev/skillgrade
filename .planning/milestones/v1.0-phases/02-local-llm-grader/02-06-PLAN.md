  <action>
In src/providers/local.ts, update the env object inside runCommand:

1. Replace `path.delimiter` with a hardcoded `":"` colon when constructing the PATH string. The shell launched by spawn is bash (via Git Bash / MSYS2), which always uses colon-separated PATH regardless of platform. Using path.delimiter returns ";" on Windows which MSYS2 ignores, causing PATH to collapse to a single entry.

2. Add `BASH_ENV: undefined` and `ENV: undefined` to the spawned env object. When the parent Node.js process has BASH_ENV set (common in CI, WSL, and developer shells), bash reads that file at startup and can reorder PATH and wipe custom vars. Passing `undefined` for a key in Node's spawn env causes that key to be absent from the child process environment — it does NOT pass through from process.env even though we spread process.env earlier.

The corrected env block should look like:
```typescript
env: {
  ...process.env,
  ...env,
  PATH: `${binDir}:${currentPath}`,
  BASH_ENV: undefined,
  ENV: undefined,
},
```

Note: `currentPath` uses `env?.PATH ?? process.env.PATH ?? ''` as before. Do not change the logic for computing currentPath, only the delimiter and the two new suppression keys.
  </action>

In tests/local-provider.test.ts, replace each bare `await fs.remove(workspace)` call (there are three, one in each test) with a small retry helper that tolerates Windows file-locking (EBUSY / EPERM from Defender or Search Indexer holding a scan lock after process exit).

Add this helper function before runTests():

```typescript
async function removeWithRetry(dir: string, retries = 5, delayMs = 200): Promise<void> {
    for (let i = 0; i < retries; i++) {
        try {
            await fs.remove(dir);
            return;
        } catch (err: any) {
            if (i === retries - 1) {
                throw err;
            }

            await new Promise(resolve => setTimeout(resolve, delayMs));
        }
    }
}
```

Replace all three `await fs.remove(workspace)` calls with `await removeWithRetry(workspace)`.

Do NOT change any assertion logic or test structure — only the cleanup calls.
